# UX Round 1 — 최기준 페르소나 사용성 진단

> **작성자**: ux-agent (최기준 페르소나 대변자)  
> **날짜**: 2026-04-28  
> **관점**: 바쁜 근무 현장의 점장 — 한 손으로 앱을 조작하며 빠른 판단이 필요한 사용자

---

## 1. Thumb Zone (엄지손가락 도달 범위) 진단

### 현황 우려사항
- 모바일 앱 특성상 **주요 액션 버튼(일정 추가, 승인/거절)이 화면 상단에 배치될 경우**, 점장이 한 손으로 조작하기 매우 어렵다.
- 현장에서 점장은 오른손에 물건을 들거나 카운터를 잡고 있는 경우가 많아, 왼손 엄지 또는 오른손 엄지만으로 앱을 조작하는 시나리오가 빈번하다.
- **Safe Thumb Zone**: 화면 하단 약 60~65% 구간. 이 영역 밖에 주요 CTA(Call To Action)가 존재하면 오조작 및 조작 포기가 발생한다.

### 페인 포인트 (최기준 시점)
> "매장 바쁠 때 한 손으로 앱 켰더니 승인 버튼이 화면 꼭대기에 있어요. 폰 떨어뜨릴 뻔했습니다."

- **위험 영역**: Navigation Bar, 상단 탭(Tab), 헤더 우상단 아이콘 등
- **개선 필요**: 핵심 액션(일정 승인, 공지 발송, 알림 확인)은 화면 하단 Safe Zone에 고정 배치 필요

---

## 2. 한 손 일정 등록 클릭 뎁스 분석

### 예상 플로우 (현재)
1. 앱 실행
2. 메인 화면 진입
3. 일정/스케줄 탭 이동
4. 날짜 선택
5. 직원 선택
6. 시간 입력
7. 저장 확인

→ **최소 7단계, 약 7번의 탭 조작 필요**

### 페인 포인트
> "점심 피크 타임에 직원이 갑자기 못 나온다고 연락왔어요. 빨리 대타 일정 넣어야 하는데 화면을 7번 눌러야 해요? 그냥 전화합니다."

- **허용 클릭 뎁스**: 긴급 액션은 **3탭 이내** 완료 가능해야 함
- **권장 개선안 제안**: 홈 화면에 "빠른 일정 등록" FAB(Floating Action Button) 또는 Bottom Sheet를 제공해 날짜·시간·직원을 한 화면에서 입력 가능하게 할 것
- **현재 플로우 문제**: 피크타임에 앱 이탈 가능성 높음 → 카카오·전화로 업무 회귀

---

## 3. 알림 환경 진단

### 현황 우려사항
- 매장 현장은 **소음이 심해 진동/소리 알림을 체감하기 어렵다**.
- 직원 휴가 요청, 출퇴근 이상, 교대 요청 등 **즉각 처리가 필요한 알림**이 묻힐 수 있음.
- 앱 내 알림 배지가 있어도 **현장 근무 중 화면을 자주 보지 않는 점장**에게는 의미가 없다.

### 페인 포인트
> "알림 왔는지 몰랐어요. 직원이 출근 안 한 걸 3시간 뒤에 알았습니다."

- **필수 요건**:
  - 긴급 이벤트(출근 미확인, 결원 발생)는 **Push Notification 강제 + 진동 반복** 방식 지원
  - 알림 목록 화면에서 **미처리 항목만 필터링**하는 기능 제공
  - 처리 완료까지 알림 배지 유지 (읽음만으로 사라지지 않도록)

---

## 4. 접근성 기준 진단

### 타이포그래피 & 대비
- 점장 사용자 연령대 고려 시(30~50대 중간 관리자), **최소 본문 16sp, 라벨 14sp** 이상 권장
- 텍스트-배경 명도 대비: **WCAG AA 기준 4.5:1 이상** 준수 필요
- 현장 조명(주방·형광등 환경)에서도 **가독성 확보**를 위한 고대비 컬러 필수

### 터치 타깃 크기
- 모든 터치 가능 요소: **최소 44×44pt (iOS HIG 기준) / 48×48dp (Material Design 기준)**
- 특히 승인/거절 버튼이 작을 경우 현장에서 오조작 발생 → **오조작 후 실행 취소(Undo) 옵션** 필수

### 색각 이상 대응
- 상태 표시(출근 완료 = 초록, 지각 = 빨강, 결근 = 회색)를 색만으로 구분하지 않도록
- **아이콘 또는 텍스트 레이블 병용** 필수
